Key Insights & Recommendations

Data-driven findings for Module 1–5 development

🔴 Dominant Disruption Type

Vehicle Breakdown accounts for 60% of all events. This should be the primary focus of Module 1 (Risk Prediction) — the model needs strong features for breakdown-type events.

🟡 Temporal Concentration

25% of events occur during rush hours (8-10 AM, 5-8 PM). Module 2 (Impact Score) should heavily weight time-of-day — a breakdown at 9 AM on ORR has 5x the impact of one at 2 AM.

🔵 Corridor Focus

62% of events occur on designated corridors. The top corridor is Non-corridor. Module 4 (Diversion) should pre-compute alternate routes for the top 10 corridors.

🟢 Resolution Performance

Median resolution time is 1.0 hours (mean: 4.9h). The large gap between median and mean suggests outliers — some events take days to resolve. Module 3 should flag these for escalation.

🟣 Road Closure Signal

Only 8.3% of events require road closure. However, road closure events likely have disproportionate impact — this is a strong binary feature for the ML model (Module 1).

🔷 Police Station Load

The busiest station is Yelahanka. Module 3 (Resource Recommendation) should balance workload across stations — current data shows uneven distribution.

🎯 Recommendations for Next Phases

Module 1 (Risk Prediction): Use event_cause + is_corridor + hour_of_day + requires_road_closure as top features. Engineer the 4-level target from priority + road_closure + resolution_time signals.
Module 2 (Impact Score): Weight rush-hour events on major corridors 3-5x higher. Use historical frequency at same junction as a multiplier.
Module 3 (Resources): Build rule matrix from the event_cause × priority cross-tab. Vehicle breakdowns need crane/towing; tree falls need BBMP crew.
Module 4 (Diversion): Pre-map top 10 corridors. ORR and Bellary Road should have pre-computed diversions ready.
Module 5 (Dashboard): Use this EDA dashboard structure as the foundation. Add real-time event feed and prediction overlays.
